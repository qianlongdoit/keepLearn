### 数据上报方案

#### 上报方式
**sendBeacon**

```javascript
window.addEventListener('beforeunload', () => {
    navigator.sendBeacon('report', JSON.stringfy(data));
})
```
+ 异步非阻塞
+ 页面、浏览器关闭后仍然可以发送
+ post请求
+ 小数据支持 < 64kb
+ 支持跨域
+ 不兼容IE


**Image打点（兜底**
使用方法
```javascript
new Image().src = `/report?data=${encodeURIComponent(JSON.stringfy(data))}`
```
### 为什么图片打底上报？
前端监控的难点在于：
  最严重的错误，往往是你“上报不了的错误”，比如：
  + 白屏
  + js崩溃
  + 内存爆掉
  + 主线程卡死

### 为什么img只能作为兜底方案
+ 只能get请求
+ url长度有限容易被缓存
+ 对js运行状态依赖极弱
+ 不受页面卸载影响（页面关闭的时候，img请求仍然可能发送出去）
+ 天然支持跨域
+ 不依赖sdk复杂逻辑

示例
``` javascript
function report(data){
    const url = 'https://monitor.xxx.com/report';

    // 首先尝试Beacon上报
    if(navigator.sendBeacon) {
        const ok = navigator.sendBeacon(url, JSON.stringify(data));
        if(ok) return;
    }

    // 其次 fetch 手动上报
    try {
        fetch(url, {
            method: 'POST',
            body: JSON.stringify(data),
            keepalive: true
        })
        return;
    } catch (e) {}

    // 最后图片兜底
    reportByImage(url, data);
}
```
